This is the only COM-touching project in the solution. `Interop/QBXMLRP2Lib.cs` is a hand-written `[ComImport]` stub with placeholder GUIDs so the solution builds with no QuickBooks SDK installed; on the QuickBooks host in Phase 9 it is replaced by a `tlbimp`-generated `Interop.QBXMLRP2Lib.dll` and this file is deleted. `RealRequestProcessor` is a Phase-1 stub that throws `NotImplementedException`; the real STA-pinned, x86 COM adapter over `QBXMLRP2.RequestProcessor` is built in Phase 2. The live-host Phase 9 qbXML validation and cleanup procedure is documented in `quickbooks/QbConnectService/docs/QBXML-REPIN.md`.
