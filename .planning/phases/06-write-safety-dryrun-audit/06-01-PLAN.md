Additional fixed details from the research:
- **`requesterId` = `"tok-" + SHA256(ApiToken).hex[..8]`** (so a token rotation shows up without ever logging the token); `"api"` if `AuthOptions.ApiToken` is blank. Bind `IOptions<AuthOptions>` into `AuditLog`. `AuthOptions` lives in namespace `QbConnectService` (file `QbConnectService/Options/AuthOptions.cs`) — the `QbConnectService.Qb.Com` project does NOT reference `QbConnectService`, so **define a tiny `AuditAuthOptions { string? ApiToken }` POCO in the `Qb.Com` `Audit/` folder and bind it from the `"Auth"` section in `Program.cs`** (don't take a cross-project dependency). (Same shape; one field; bound from the same section.)
- **Canonical bytes** for the hash: `Utf8JsonWriter` writing fields in this HAND-CODED order, then compute SHA-256 of those bytes, then append `hash` as the 11th property when writing the line: `seq, timestampUtc, op, args, qbXmlRequest, responseStatusCode, responseStatusSeverity, responseStatusMessage, requesterId, prevHash` → then `hash`. Do NOT key-sort `args` (serialize it as-sent via `JsonSerializer.Serialize(writer, rec.Args)`). `VerifyChainAsync` re-runs the *same* `ComputeRecord` helper on the parsed fields (not the on-disk byte layout) so it's robust to whitespace.
- **Genesis `prevHash` = 64 zeros** (a fixed `const string`, not `null` — easier to test).
- **Hex:** `Convert.ToHexString(SHA256.HashData(bytes)).ToLowerInvariant()` (.NET 8 — do NOT use `Convert.ToHexStringLower`, that's .NET 9+; the project targets `net8.0-windows`).
- **Timestamp:** `DateTime.UtcNow.ToString("O", CultureInfo.InvariantCulture)` (ISO-8601, `Z` suffix). Never local time.
- **`AuditLog` is a singleton** with a `SemaphoreSlim(1,1)` around the whole append (read-cached-last-hash → compute → write), cached `_lastSeq`/`_lastHash` in memory after first load (`EnsureLoadedAsync` reads the file once). `VerifyChainAsync` also takes the gate.
- **Dir-create:** `Directory.CreateDirectory(Path.GetDirectoryName(_filePath)!)` before the first append (`File.AppendAllText` creates the file but not parent dirs). **Path fallback:** if `Audit:Path` is blank → `Path.Combine(Path.GetTempPath(), "QbConnectService", "audit")` (mirrors `QbResponseSpiller`'s spill fallback).
- **403 at the ops endpoint** is produced via `Results.Problem(403, ...)` *directly* (the endpoint statically knows it's a write op + writes are off — no exception round-trip). The *manager's* defensive check is the one that throws `QbWriteForbiddenException` (its caller is "whoever called `ExecuteAsync`").
- **`QbWriteForbiddenException`** is a plain `Exception` subclass (NOT a `QbException` subclass — that family maps to 503; this one maps to 403). Put it in `QbExceptions.cs` (namespace `QbConnectService.Qb`).
- **`ApiExceptionHandler`** detail line already does `ex.Message` for non-`QbException` — so the new arm just adds `QbWriteForbiddenException => (StatusCodes.Status403Forbidden, "Writes disabled")` before the generic `_ =>`.
- **`appsettings.sample.json`** already has `"Audit": { "Path": "..." }` — just confirm `AuditOptions` binds it; no edit needed (a one-line comment is fine but optional). Keep config minimal — do NOT add an `Audit:Enabled` flag.

    public virtual Task<DryRunResult> DryRunAsync(IReadOnlyDictionary<string, object?> args, CancellationToken ct = default)
        => Task.FromResult(new DryRunResult(BuildRequest(args), $"{Name}: would send the qbXML below.", Array.Empty<PreFlightCheck>(), new Dictionary<string, object?>(), _safety.AllowWrites));

    public override async Task<object?> RunAsync(IReadOnlyDictionary<string, object?> args, CancellationToken ct = default)
    {

            ["rows"] = parsed.First.Rows,
            ["auditSeq"] = seq,
            ["rawSpilledTo"] = parsed.RawSpilledTo,
        };

            .Select(k => $"{k}: '{before.GetValueOrDefault(k)}' -> '{after.GetValueOrDefault(k)}'")
