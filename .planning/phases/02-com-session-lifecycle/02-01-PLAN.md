<objective>
Build the COM session lifecycle layer for the QuickBooks integration: `QbConnectionManager` (long-lived singleton owning `OpenConnection -> BeginSession -> ProcessRequest -> EndSession -> CloseConnection`), a dedicated STA worker thread that all `IRequestProcessor` calls run on, a `SemaphoreSlim(1,1)` serialization gate with a bounded "busy" wait, a watchdog timeout that poisons a wedged request, a dead-ticket reconnect that rebuilds the COM object and retries exactly once, the `QbErrors` HRESULT-to-message map, the real `RealRequestProcessor` adapter over Phase 1's `QBXMLRP2Lib.cs` placeholder interop, and the DI wiring in `Program.cs`. Everything is built and tested against `FakeRequestProcessor` — Phase 2 needs no QuickBooks installed.

Purpose: Phase 2 is the runtime substrate every later phase (qbXML engine, read/write ops, REST API) calls into. It must be concurrency-correct (one STA thread, one in-flight call, no continuation-on-pump deadlocks) and COM-lifetime-correct (rebuild-don't-revive, FinalRelease-on-the-right-thread). Maps to requirements SESS-01..SESS-05.

Output: New files in `QbConnectService.Qb.Com` (`QbOptions.cs`, `QbErrors.cs`, `QbExceptions.cs`, `StaThread.cs`, `QbConnectionState.cs`, `QbConnectionManager.cs`), a rewritten `RealRequestProcessor.cs`, DI wiring in `Program.cs`, new keys in `appsettings.sample.json`, small extensions to `FakeRequestProcessor.cs`, and new test files (`QbOptionsBindingTests.cs`, `QbErrorsTests.cs`, `StaThreadTests.cs`, `QbConnectionManagerTests.cs`, `RealRequestProcessorSmokeTests.cs`) plus an updated `HostStartupTests.cs`.
</objective>

**Deliberate deviations / known limits (so the reviewer doesn't flag them):**
- **Hand-rolled single retry, NOT Polly.** `STACK.md` lists Polly v8 "for the one retry," but the retry must run bespoke teardown + rebuild + re-open between the two attempts (not "call the same delegate again"), so a `try/catch` is clearer and avoids adding the Polly package. Phase 2 adds **zero** NuGet packages. This is an intentional, justified deviation from `STACK.md`.
- **Dead-ticket set = just `0x8004040D` for now.** Conservative. Widen in Phase 9 with on-box evidence, not speculatively. `QbErrors.DeadTicket` is the single place to widen it.
- **`OpenMode=DoNotCare` / `ConnectionType=LocalQBD` are hard-coded in `QbConnectionManager`** (SESS-01 is a requirement, not a config knob).
