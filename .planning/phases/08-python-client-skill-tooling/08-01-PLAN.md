   - Module docstring noting it's the HTTPS client for `QbConnectService` (pure HTTP, no QuickBooks dependency), every request sends `Authorization: Bearer <token>`, and the conservative retry policy.
   - `class QbApiError(RuntimeError)` with `status_code`, `title`, `detail`, `qb_error_code`, `body` attributes; `__init__(self, status_code, title=None, detail=None, qb_error_code=None, body=None)`; builds a readable message (`"{status} {title}: {detail} [{qb_error_code}]"`).
   - `class QbClient`:
     - `__init__(self, base_url, token, *, verify_tls=True, timeout=60.0, retries=3, session=None)` — validates `base_url`/`token` non-empty (`ValueError`), `self.base_url = base_url.rstrip("/")`, sets `Authorization: Bearer <token>` on a `requests.Session`, mounts an `HTTPAdapter(max_retries=Retry(total=retries, connect=retries, read=retries, backoff_factor=0.5, status_forcelist=(502,503,504), allowed_methods=frozenset({"GET"}), raise_on_status=False))` on `https://` and `http://`.
     - `@classmethod from_env(cls, *, dotenv_path=None)` — `load_dotenv(dotenv_path)` (import inside, no-op if `python-dotenv` missing or file missing; `dotenv_path=None` lets dotenv do its default upward walk); reads `QB_API_BASE_URL` + `QB_API_TOKEN` (raise `RuntimeError` if either missing — message names both and "in the environment or quickbooks/clients/.env"); `QB_VERIFY_TLS` → `False` for `0/false/no/off`, `True` for `1/true/yes/on/""`, otherwise the raw string (a CA-bundle path); `QB_TIMEOUT`/`QB_RETRIES` parsed to float/int with the defaults.
     - `health(self) -> dict` → `GET /api/health`, returns the parsed JSON dict.
     - `ops(self) -> list[str]` → `GET /api/ops`, returns `["ops"]`.
     - `qbxml(self, raw_xml: str) -> str` → `POST /api/qbxml` with `data=raw_xml.encode("utf-8")`, `Content-Type: application/xml`, `timeout`, `verify`; raises for status; returns `r.text`. (Non-retrying — `/api/qbxml` may contain a write.)
     - `op(self, name: str, args: dict|None = None) -> Any` → `POST /api/ops/{name}` with `json=args or {}`, `timeout`, `verify`; raises for status; returns `r.json()["result"]`. (Non-retrying — a write op is not idempotent.)
     - `dryrun(self, name: str, args: dict|None = None) -> dict` → `POST /api/ops/{name}/dryrun` via a one-shot retrying call (`/dryrun` has zero side effects per spec §6 — its own `Session` with `allowed_methods=frozenset({"GET","POST"})`, same headers + `verify`); raises for status; returns `r.json()["dryRun"]`.
     - `_get(path)` helper (GET with `timeout`/`verify`, raise-for-status, return `.json()`); `_post_retryable(url, *, json)` helper for `dryrun`; `_raise_for_status(r)` static — if `r.status_code < 400` return; else parse `title`/`detail`/`qbErrorCode` from the body when `Content-Type` is `json`/`problem+json` (tolerate non-JSON bodies — keep `body = r.text`), raise `QbApiError(r.status_code, title, detail, qb, body)`.
   - Top-of-module constants: `DEFAULT_TIMEOUT = 60.0`, `DEFAULT_RETRIES = 3`.
   - Use `from __future__ import annotations`. No hard-coded URLs or tokens anywhere. Never call `requests.get`/`post` without both `timeout=` and `verify=`.
